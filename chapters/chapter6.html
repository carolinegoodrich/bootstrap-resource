CHAPTER SIX:

Our Final Project, Part 2

In this chapter we will focus on creating the styles for our web pages, incorporating some native to bootstrap and some that come from our head, as well as make our website mobile-responsive. The shining appeal of Bootstrap, is the fact that it makes web pages and web applications mobile responsive just by using classes and divs, instead of the old way of using a separate stylesheet to control the mobile styles.

We’re going to break this chapter into two mini chapters! Aren’t you excited?? If only you could answer!


Mini Part 1: Mobile Styles

To be honest with you guys, this is still focusing on HTML. I kind of lied. But, sometimes in order to get the style results you want, you have to add additional styles to these tags. So because of that, I’m putting it in the style section. Don’t complain, I don’t wanna hear it.

I’m giving you free reign here in regards to how you want your site to look! My pages and examples are merely suggestions to show you how these styles can be applied. If you don’t like way I do it, too bad I’m the best out there.

Bootstrap makes mobile styling easy because it utilizes the class tags in the column system. You specify how many columns you want that particular object to fill depending on the screen size: xs, sm,md,lg.

Please see below for an explanation of the grid system and correlating screen sizes.


I’m going to add column specifications to my index.html page where I have the page explanation sections. Click below to download an updated version of my index.html page to see the alterations I’ve made.

Notice that when you change the screen size of your preview window your elements shift depending on your code. You can also add elements that are hidden at a certain screen size and then appear on other screen sizes. This is how hidden menus work! Check out the GetBootstrap link in the resource page to learn how to make your menu a bar menu on xs screens

Mini Part 2: Style

We’re going to have two different stylesheets, just to get experience with importing multiple external stylesheets for one website. As most of the other sections, you can tweak this to appeal more to your design taste.

my stylesheet for the index page is going to be called main-styles. These will also contain body padding to keep the body from being overrun by the menu. The other pages will have a similar layout controlled by another stylesheet named other-styles.

In the resource page, you will find links to design principle articles and how to choose the right color scheme. I try to either use monochromatic colors or complimentary colors. I also like websites that ulitize white space, because I feel it makes the user be able to focus on individual sections instead of being overwhelmed. Good design takes some time to master, which is why there’s a whole field dedicated to it (User Experience, or UX Design.)

Because my site is focusing on manatees, I’m going to use variations of blue on my site. I’m going to use the keyword btn-info in order to style my buttons, which uses a Bootstrap keyword. Another keyword that we’ve utilized is the img-circle keyword, which makes our thumbnail images round without excessive CSS corner radius attributes.
